# 02dce21 — fix: bundle internal workspace packages for publish

## Target file(s)
- package.json (exists, but does not use bundledDependencies / @gsd packages)
- scripts/validate-pack.js (exists)

## Divergence
Diverged. Upstream gsd-pi ships internal `@gsd/*` workspace packages inside the root tarball via `bundledDependencies`. Otto-cli does not use this model — it pins native engine packages via `optionalDependencies` (`@cmetech/otto-engine-*`) and publishes them as separate npm packages from CI (per the documented otto release flow). Otto-cli's root `package.json` has no `@gsd/*` entries and no `bundledDependencies`. The guard rewrite in `validate-pack.js` is checking for "every shipped @gsd/* dependency must appear in `bundledDependencies`" — a constraint that does not exist in otto's publish pipeline.

## Concrete edits
None.

## Verdict
Do-not-port. The change addresses a publish-architecture concern (bundling internal workspace packages) that otto-cli already solves a different way (separate per-arch npm packages with version-sync test). Porting the validate-pack guard would generate false failures or do nothing useful.

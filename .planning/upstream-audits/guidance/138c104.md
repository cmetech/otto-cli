The native build is already separated upstream of npm-publish in otto, so "publish prod native packages inline" is conceptually the opposite of otto's architecture.

## Verdict
Do-not-port. The change re-architects upstream's two-workflow pipeline to inline prod native publishes into the npm-publish prod-release job. Adopting the upstream changes here would require ripping out otto's working tag-driven release flow.

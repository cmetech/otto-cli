## Divergence
Otto-cli has the same three core files as gsd-pi: `src/headless-answers.ts` (with `AnswerInjector` + `loadAndValidateAnswerFile`), `src/headless.ts`, and `src/resources/extensions/claude-code-cli/stream-adapter.ts`. The upstream fix (a) adds a `normalizeQuestionId` + `findConfiguredQuestionAnswer` fuzzy matcher to `headless-answers.ts` so headless answer files can use loose key names (case/dash/underscore + `_confirm` suffix stripping), (b) propagates the answer-file path via env var to the child process in `headless.ts`, and (c) adds a parallel header-side answer lookup in stream-adapter.ts so claude-code-cli elicitation forms can also be auto-answered. The key naming divergence: upstream uses `GSD_HEADLESS_ANSWERS_PATH`; otto-cli uses `OTTO_HEADLESS` (confirmed at `src/headless.ts:464` and `stream-adapter.ts:1288`), so the env var must be renamed to `OTTO_HEADLESS_ANSWERS_PATH` on port. The interactive-mode commit also stops calling `loadAndValidateAnswerFile` twice — otto-cli already does the same.

Manual rather than cherry-pick because of the env-var rename (`GSD_` → `OTTO_`) and because the matching tests must be rewritten against otto's test layout.

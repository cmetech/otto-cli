# 3f5e830 — fix(issue): getRemoteUrl swallows transient git failures, flipping identity hash

## Target file(s)
- src/resources/extensions/workflow/repo-identity.ts
- src/resources/extensions/workflow/tests/repo-identity-worktree.test.ts

## Divergence
Otto's `repo-identity.ts` is at the pre-fix baseline: `getRemoteUrl()` returns `string` and yields `""` on any failure, exactly matching the bug being fixed. The neighbouring helpers (`readGsdIdMarker`, `repoIdentity`, `externalGsdRoot`, `ensureGsdSymlinkCore`, `writeRepoMeta`) all exist with the same shapes. The only path-level divergence is the `extensions/gsd/` → `extensions/workflow/` rename and the otto rebrand string in the file header. There is no fork-specific edit to the identity logic itself.

## Concrete edits
1. Change `function getRemoteUrl(basePath: string): string` to return `string | null`, and replace the `catch { return "" }` with `catch (error) { console.warn("[GSD] repo-identity: failed to resolve remote.origin.url for ...", error); return null; }`. (Keep the otto log prefix `[OTTO]` if you prefer — match neighbouring `logWarning` calls in the file.)
2. In `repoIdentity(basePath)`, after `const remoteUrl = getRemoteUrl(basePath); if (remoteUrl) { ... }`, add the `if (remoteUrl === null) { const markerRoot = resolveGitRoot(basePath); const markerId = readGsdIdMarker(markerRoot); if (markerId) return markerId; }` recovery block before falling through to the local-only hash branch.
3. In `externalGsdRoot`, replace the inline construction with the `resolveExternalPathWithRecovery(basePath).path` indirection if otto already has the helper (verify — gsd-pi adds it in the same commit family). If otto does not yet have `resolveExternalPathWithRecovery`, leave this part out and file a follow-up.
4. In `ensureGsdSymlinkCore`, change `writeRepoMeta(externalPath, getRemoteUrl(projectPath), resolveGitRoot(projectPath));` to `writeRepoMeta(externalPath, getRemoteUrl(projectPath) ?? "", resolveGitRoot(projectPath));` so the function signature change is non-breaking.
5. Port the new tests from `tests/repo-identity-worktree.test.ts` (paths under `extensions/workflow/tests/`); they exercise transient-failure recovery via the marker file and need no further rename beyond `gsd` → `workflow` in path strings.

## Verdict
Worth porting — the bug (transient `git config` failure flipping a project's identity hash and triggering first-time-init) is severe and the otto file is at the exact pre-fix state. Conflict risk is low; the only manual step is the path/rebrand rename and verifying `resolveExternalPathWithRecovery` exists. If it doesn't, drop edit #3 — the core fix (edits #1, #2, #4) stands alone.

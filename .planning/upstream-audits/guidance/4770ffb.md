# 4770ffb — fix(bug-2): Uncaught exception guards exit without releasing auto-mode locks

## Target file(s)
- src/resources/extensions/workflow/auto-supervisor.ts
- src/resources/extensions/workflow/tests/crash-handler-secondary.test.ts
- src/resources/extensions/workflow/tests/signal-handlers.test.ts

## Divergence
Otto's `auto-supervisor.ts` keeps the upstream `CLEANUP_SIGNALS: NodeJS.Signals[] = ["SIGTERM", "SIGHUP", "SIGINT"]` (line 14) and uses it identically (`process.on/off` over the array on lines 39, 44, 57, 65). The signal-list extension to include `SIGBREAK` applies verbatim. The companion crash-handler change is in `tests/crash-handler-secondary.test.ts` and `tests/signal-handlers.test.ts`, both of which exist in otto. The crash-handler fix is the substantive one: gsd-pi's `_gsdEpipeGuard` calls `process.kill(process.pid, "SIGTERM")` before `process.exit(1)` so the registered SIGTERM cleanup path runs and releases auto-mode locks before termination. This implementation detail likely lives in otto's `register-extension.ts` or wherever `_gsdEpipeGuard` is defined — verify and port the `process.kill(pid, "SIGTERM")` call ahead of the existing `process.exit(1)`.

## Concrete edits
1. In `src/resources/extensions/workflow/auto-supervisor.ts`, change the `CLEANUP_SIGNALS` array literal on line 14 from `["SIGTERM", "SIGHUP", "SIGINT"]` to `["SIGTERM", "SIGHUP", "SIGINT", "SIGBREAK"]`. No other code in this file needs touching — every loop already iterates the array.
2. Locate `_gsdEpipeGuard` (grep `_gsdEpipeGuard` and `_ottoEpipeGuard` across `src/resources/extensions/workflow/`). In its unrecoverable-error branch, before `process.exit(1)`, insert `try { process.kill(process.pid, "SIGTERM"); } catch { /* fall through to exit */ }`. The guard must still synchronously call `process.exit(1)` after — the SIGTERM handler may not get a chance to halt the process.
3. Update `tests/signal-handlers.test.ts` to add `SIGBREAK` to the before/after counters and the assertion for "+1 listener". Mirror the pattern already used for `SIGINT`.
4. Update `tests/crash-handler-secondary.test.ts` to monkey-patch `process.kill` alongside `process.exit`, asserting the guard issues a SIGTERM (`killSignal === "SIGTERM"`) before exit code 1, and accepting either intercept in the throws-matcher (`/process\.(kill|exit) intercepted/`).
5. If otto's epipe guard symbol is renamed (`_ottoEpipeGuard` etc.), keep the gsd-pi name only in tests that look up by `.name`; adjust accordingly.

## Verdict
Worth porting. The bug — uncaught-exception crash bypassing lock release, leaving the next auto-mode invocation wedged on a stale lock — is real and otto inherits the exact same lock-release-via-SIGTERM-handler architecture. The signal-array bump alone (edit #1) is a one-liner with no risk; the crash-handler edit (#2) requires finding otto's guard symbol but the change is structurally identical. Test ports follow naturally.

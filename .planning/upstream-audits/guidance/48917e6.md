# 48917e6 — fix: resolve Windows npm global bin path

## Target file(s)
- scripts/install/npm-global.js (does not exist in otto)
- scripts/install/prereqs.js (does not exist in otto)

## Divergence
gsd-pi has a `scripts/install/` subdirectory containing `npm-global.js` (returns `{ prefix, root, binDir, packageRoot }`) and `prereqs.js` (returns the global bin dir). The fix makes `binDir` Windows-aware: on win32, the npm prefix *is* the bin dir; elsewhere it's `prefix/bin`. Otto has no `scripts/install/` subdir at all. Its install model uses a managed bin under `~/.otto/agent/bin` (`scripts/install.js` line 167, 520, 523) rather than npm's global bin tree. None of the files this commit patches exist in otto, and the otto install pipeline does not look up `npm prefix -g`'s bin dir.

## Concrete edits
1. None.

## Verdict
Skip. The bug fixed (Windows users couldn't locate the gsd binary because the path joined a redundant `bin` segment) doesn't exist in otto's install architecture. If otto ever adopts the npm-global-bin model, port at that time.

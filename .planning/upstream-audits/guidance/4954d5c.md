# 4954d5c — fix: avoid OOM when seeding openai in validate-pack global smoke

## Target file(s)
- scripts/validate-pack.js

## Divergence
gsd-pi's `validate-pack.js` global-smoke step used to `npm install openai --ignore-scripts --no-save` inside the *globally installed* package tree (`globalRoot`), which OOMed in CI while resolving the entire transitive graph. The fix replaces the install with a `cpSync` from the *local* tarball install's already-resolved `node_modules/openai`. Otto's `validate-pack.js` follows the same skeleton (lines 149, 161, 164, 184, 185) using `installedRoot = join(installDir, 'node_modules', '@cmetech', 'otto')` and a `loaderPath` smoke. Verify whether the otto smoke also reaches for `openai` in the global tree (grep `openai` in `validate-pack.js`). If yes, the fix is a clean port: import `cpSync` from `node:fs` and replace the npm install block with the cp-from-local-install pattern. The scope name change (`@gsd/pi-ai` → `@cmetech/otto`) does not affect this step because the missing dep being seeded is the third-party `openai` package, not the workspace bundle.

## Concrete edits
1. Add `cpSync` to the `fs` import at the top of `scripts/validate-pack.js`.
2. Locate the `try { ... npm install openai --ignore-scripts --no-save ... }` block (around line 486 in gsd-pi; find the equivalent in otto by grepping `openai`). If present, replace with:
   ```js
   const localOpenaiDir = join(installedRoot, 'node_modules', 'openai');
   const globalOpenaiDir = join(globalRoot, 'node_modules', 'openai');
   if (!existsSync(join(globalOpenaiDir, 'index.js')) && existsSync(join(localOpenaiDir, 'index.js'))) {
     mkdirSync(join(globalRoot, 'node_modules'), { recursive: true });
     cpSync(localOpenaiDir, globalOpenaiDir, { recursive: true });
   }
   ```
3. If otto's smoke doesn't seed `openai` at all, skip this commit — the OOM scenario doesn't apply.

## Verdict
Worth porting if (and only if) otto's `validate-pack.js` performs the same global-install-then-seed-openai dance. Otto's bundling otto-side may already vendor `openai` differently — confirm before changing. The fix itself is mechanical and low-risk; the contingent check is whether otto reaches the OOM site at all. Severity is `CRITICAL_STABILITY` in the audit because OOM kills the CI smoke completely.

# 4ba1821 — fix: stabilize pack install integration test

## Target file(s)
- src/tests/integration/pack-install.test.ts

## Divergence
gsd-pi rewrites `runNpmQuiet` to capture stdout/stderr on failure and re-throw a richer Error containing exit status, signal, and the last 4KB of both streams. It also adds a `runNodeScript` helper and an `isAbsolute` import. The motivation: flaky CI failures in pack-install were producing inscrutable `Command failed` messages with no diagnostic context. Otto's `pack-install.test.ts` already has the same `runNpmQuiet(args: string[], sandbox: NpmSandbox): void` signature (note: void return) with the same call sites for `pack`, `install --prefix`, etc. So the change is a clean structural upgrade — return `string` instead of `void` and wrap in try/catch with a rich Error.

The companion `runNodeScript` helper appears to be added for a separate gsd-pi test case (not visible in the diff I inspected). Verify whether otto's pack-install test needs it; if it does not, omit.

## Concrete edits
1. In `src/tests/integration/pack-install.test.ts`, add `isAbsolute` to the `node:path` import.
2. Add a `formatCommand(args: string[]): string` helper that returns `"npm " + args.join(" ")`.
3. Change `runNpmQuiet(args: string[], sandbox: NpmSandbox): void` to return `string`, wrap the `execFileSync` call in try/catch, and on catch throw:
   ```ts
   const stdout = Buffer.isBuffer(err.stdout) ? err.stdout.toString("utf-8") : (err.stdout ?? "");
   const stderr = Buffer.isBuffer(err.stderr) ? err.stderr.toString("utf-8") : (err.stderr ?? "");
   throw new Error([
     formatCommand(args) + " failed",
     "status=" + (err.status ?? "<none>") + " signal=" + (err.signal ?? "<none>"),
     stdout.trim() ? "stdout:\n" + stdout.trim().slice(-4000) : "",
     stderr.trim() ? "stderr:\n" + stderr.trim().slice(-4000) : "",
   ].filter(Boolean).join("\n\n"));
   ```
4. If gsd-pi's commit also adds the `runNodeScript(relativePath: string)` helper and a test case that uses it, port both. If the helper is unused in otto's test scenarios, skip it.
5. Confirm no call site of `runNpmQuiet` relied on the void return (none do per the gsd-pi pattern; the change is forward-compatible).

## Verdict
Worth porting — it's a pure test-diagnostics improvement that costs nothing and pays off the first time pack-install flakes on macOS or Windows CI. Otto's pack-install.test.ts shape matches gsd-pi's enough that the port is mechanical. The `runNodeScript` addition is contingent on whether otto's test needs it.

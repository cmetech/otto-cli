# 54a4434 — fix: restore workspace:* root deps after prepack regression

## Target file(s)
- package.json (root)

## Divergence
Not applicable. Upstream uses pnpm `workspace:*` protocol in root `package.json` dependencies and runs a `prepack` hook to rewrite them to semver ranges for npm publish. A prior commit (3181463) left the semver ranges in place after publish, breaking `pnpm install --frozen-lockfile` in CI. Otto-cli's root `package.json` does NOT use `workspace:*` for any dependency (verified `grep workspace: package.json` returns empty). Otto already uses pinned semver in `optionalDependencies` for the `@cmetech/otto-engine-*` packages (lines 205-209) and does not appear to have a `prepack`-based rewrite flow.

## Concrete edits
None.

## Verdict
Do-not-port. The regression that this commit fixes never existed in otto because otto doesn't use the `workspace:*`-then-rewrite pattern. Otto's release flow (per memory: trusted-publishing via build-native.yml + npm-publish.yml, version-sync.test asserts pin parity) achieves the same end state with a different, simpler mechanism.

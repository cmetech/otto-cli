# 584f887 — fix(bug-1): Fresh install is non-functional

## Target file(s)
- scripts/install.js
- scripts/postinstall.js
- src/tests/postinstall.test.ts

## Divergence
Near-identical. Otto's `scripts/install.js` has the exact same buggy line — `const IS_POSTINSTALL = !!process.env.npm_lifecycle_event` at line 31 — which treats ANY npm lifecycle event (including run/test/start) as if it were a postinstall. Otto's `scripts/postinstall.js` line 10 also uses the same `process.env.npm_lifecycle_event = process.env.npm_lifecycle_event || 'postinstall'` shim. `src/tests/postinstall.test.ts` exists in otto. The only s/gsd/otto/ rename is the env-var name (`GSD_POSTINSTALL` → `OTTO_POSTINSTALL`).

## Concrete edits
1. In `scripts/install.js` (line 31), replace:
   ```js
   const IS_POSTINSTALL = !!process.env.npm_lifecycle_event
   ```
   with:
   ```js
   const IS_POSTINSTALL =
     process.env.npm_lifecycle_event === 'postinstall' ||
     process.env.OTTO_POSTINSTALL === '1'
   ```
2. In `scripts/postinstall.js`, after the existing `npm_lifecycle_event` shim line, add:
   ```js
   process.env.OTTO_POSTINSTALL = '1'
   ```
3. In `src/tests/postinstall.test.ts`, add the upstream regression test (verifies install.js contains both guarded predicates), substituting `OTTO_POSTINSTALL` for `GSD_POSTINSTALL` in the assertion regex.

## Verdict
Cherry-pick. Same release-blocker class of bug as 5754851 (fresh install non-functional), independent root cause. The fix is mechanical and small — apply both 5754851 and 584f887 together since they jointly resolve "fresh install unusable" reports. Recommend a focused commit pair landing in the next patch release.

# 59ec800 — fix(coding-agent): bypass age gates for self-update

## Target file(s)
packages/pi-coding-agent/src/config.ts

## Divergence
Heavily diverged. Upstream edits `getSelfUpdateCommandForMethod` / `makeSelfUpdateCommandStep` to inject release-age flags into pnpm/bun/npm install arg arrays. otto-cli's config.ts has NO such command-builder infrastructure: `getUpdateInstruction` simply returns plain strings like `Run: pnpm install -g ${packageName}` (no `--ignore-scripts`, no `--prefix`, no arg arrays). otto-cli also has extra install methods (`bun-binary`, `yarn`). The upstream functions/tests do not exist here. NOT cherry-pickable.

## Concrete edits
- In packages/pi-coding-agent/src/config.ts `getUpdateInstruction` (~lines 52-66), append the corresponding minimum-release-age flag to each method's instruction string:
  - pnpm: `pnpm install -g --config.minimumReleaseAge=0 ${packageName}`
  - bun: `bun install -g --minimum-release-age=0 ${packageName}`
  - npm: `npm install -g --min-release-age=0 ${packageName}`
- Decide otto-cli policy for `--ignore-scripts` (upstream includes it; otto-cli currently omits it — leave as-is unless intentionally aligning).
- If otto-cli ever gains a structured self-update command builder, mirror the same flags there.

## Verdict
manual-port — intent is clear (add release-age-bypass flags) but otto-cli's self-update surface is a different shape (plain instruction strings, no arg arrays); apply flags by hand to each method string.

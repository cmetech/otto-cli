Otto's release flow is structurally different: `syncVersionSurfaces` in `scripts/lib/version-sync.cjs` (lines 125–132, verified) **already pins** root `optionalDependencies[@cmetech/otto-engine-*]` to the new version inline, in the same atomic call that `bump-version.mjs` makes BEFORE the lockfile regen step. So otto's lockfile is correct on the first `npm install --package-lock-only` pass. Otto's `version-sync.cjs` also already includes the verifier loop (lines 209–216) that upstream is adding here. Otto's `build-native.yml` does not run `npm ci --omit=optional` at all — it publishes platform packages without installing the root tree, so the CI half of the patch doesn't apply.

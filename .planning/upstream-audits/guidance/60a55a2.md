## Divergence
Diverged. otto's edit-diff.ts is a hand-rolled pure-JS differ (`buildLineDiff`/`generateDiffString`) and deliberately does NOT import the `diff` library (header comment cites issue #453 about avoiding native/heavy diff in preview paths). So upstream's `generateUnifiedPatch` (which uses `Diff.createTwoFilesPatch` + `Diff.FILE_HEADERS_ONLY`) has no existing `Diff` import to hang off. Good news: the `diff` package IS a dependency (package.json: "diff": "^8.0.2", "@types/diff": "^7.0.2"), so it can be imported. edit.ts has `EditToolDetails` and calls `generateDiffString` at ~line 205 but currently sets only `details: { diff }` (no `patch`).

## Concrete edits
1. edit-diff.ts: add an import for the diff lib (e.g. `import { createTwoFilesPatch, FILE_HEADERS_ONLY } from "diff";` — VERIFY `FILE_HEADERS_ONLY` is exported by diff@8; if not, drop `headerOptions` or use the available equivalent). Add `export function generateUnifiedPatch(path, oldContent, newContent, contextLines = 4)` calling `createTwoFilesPatch(path, path, oldContent, newContent, undefined, undefined, { context: contextLines, headerOptions: FILE_HEADERS_ONLY })`. Note: this reintroduces a `diff`-lib dependency into a file the comment says should stay native-free — patch generation runs in the execute path (edit.ts), not the preview render path, so this is acceptable, but call it out.
2. edit.ts: import `generateUnifiedPatch`; add `patch: string` to `EditToolDetails`; after `generateDiffString(...)` compute `const patch = generateUnifiedPatch(path, baseContent, newContent);` and include `patch` in the `details` object passed to the result. Verify the variable names (`baseContent`/`newContent`/`path`) match otto's local scope at ~line 205.
3. Port the tools.test.ts assertions (patch contains `--- `/`+++ `/`@@`/`-Hello, world!`/`+Hello, testing!` and `applyPatch(...)` round-trip); add `import { applyPatch } from "diff";`.

## Verdict
manual-port — additive and low-risk, but requires introducing a `diff`-lib import into edit-diff.ts and verifying `FILE_HEADERS_ONLY` exists in diff@8.

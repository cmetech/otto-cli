# 6bef491 — fix(gsd): ignore nested SvelteKit types imports

## Target file(s)
- src/resources/extensions/workflow/post-execution-checks.ts
- src/resources/extensions/workflow/tests/post-execution-checks.test.ts

## Divergence
Otto's `post-execution-checks.ts` at line ~324 already short-circuits `^\.{1,2}/\+types/` imports (React Router generated types), but it does NOT have the corresponding `$types` (SvelteKit) branch the upstream had pre-patch. Otto needs to add the `$types` branch entirely (with the wider regex) if SvelteKit projects are supported targets.

3. Port the new tests from upstream's `tests/post-execution-checks.test.ts` (22 added lines) — they exercise nested `$types` paths.

# 7448d89 — fix: stabilize tool invocation matching

## Target file(s)
- packages/pi-coding-agent/src/modes/interactive/components/tool-execution.ts
- packages/pi-coding-agent/src/core/chat-controller-ordering.test.ts

## Divergence
Otto carries pi-coding-agent's interactive `tool-execution.ts` and `chat-controller-ordering.test.ts` at identical paths. The bug is in `matchesInvocation`: it uses `JSON.stringify(this.args)` for equality, which is key-order-sensitive — so the same logical call with `{ command, timeout }` vs `{ timeout, command }` mismatches and prevents merging `tool_execution_start` with the corresponding `message_update`. The fix adds a `stableJsonStringify` helper that sorts object keys before serializing.

## Concrete edits
1. In `packages/pi-coding-agent/src/modes/interactive/components/tool-execution.ts`, add the `stableJsonStringify(value: unknown): string` function from upstream (above `ToolExecutionOptions` interface, around current line 295).
2. In `matchesInvocation`, replace `JSON.stringify(this.args ?? null) === JSON.stringify(args ?? null)` with `stableJsonStringify(this.args ?? null) === stableJsonStringify(args ?? null)`.
3. In `packages/pi-coding-agent/src/core/chat-controller-ordering.test.ts` around line 1156, mirror the upstream test edits: the `toolBlock.arguments` should be `{ timeout: 30, command: "git status" }` and the `tool_execution_start.args` should be `{ command: "git status", timeout: 30 }` — purposely different key order to exercise the new stable comparison.

## Verdict
cherry-pick — paths and surrounding code align 1:1, no rename, no upstream-only seams. Fixes a real UI merging bug for any model that emits args with non-deterministic key order. Apply as-is.

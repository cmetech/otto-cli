# 784be52 — fix(publish): remove @gsd/* from root dependencies to fix EUNSUPPORTEDPROTOCOL on install

## Target file(s)
- none

## Divergence
The bug fixed here: gsd-pi's root `package.json` declared seven `@gsd/*: workspace:*` dependencies that leaked to the published npm packument because `npm publish` builds the manifest from `package.json` *before* the `prepack` hook strips them, so consumers got `EUNSUPPORTEDPROTOCOL` on `npm install -g @opengsd/gsd-pi`.

Otto-cli does NOT declare any `@gsd/*` or `workspace:*` dependencies in its root `package.json` — confirmed: the only workspace-shaped deps are `@cmetech/otto-engine-*` and those are `optionalDependencies` pinned to explicit versions (1.3.2), not workspace ranges. There is nothing to strip. The companion `scripts/validate-pack.js` workspace-leak guard is also gsd-pi-internal and otto has its own validate-pack pipeline.

## Concrete edits
None.

## Verdict
do-not-port — otto's publish surface was designed without the workspace:* root-dep antipattern. The bug class doesn't exist here, and the validate-pack guard targets the gsd-pi-specific tarball layout.

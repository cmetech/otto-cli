## Divergence
This is a bundle of seven CodeQL-driven hardening fixes. Otto's targets exist for five of seven:

1. `pi-ai/src/providers/openai-completions.ts` — otto has the file, with the same `baseUrl.includes("api.openai.com")` (lines 81, 759, 762–772) and the same provider-detection chain. The patch replaces `.includes(host)` with `new URL(baseUrl).hostname === host` / `.endsWith(suffix)` via small `hasHostname`/`hostnameEquals`/`hostnameEndsWith` helpers, eliminating substring-collision spoofing (e.g., `evil-api.openai.com.attacker.io`). Applies cleanly with no rename.

2. `pi-coding-agent/src/core/package-manager.ts` — otto has the file. Adds `assertSafeGitArg`/`assertSafeGitRepo`/`assertSafeGitRef` and wraps `git clone`/`git checkout`/`git fetch` args. Applies cleanly.

3. `pi-coding-agent/src/core/tool-ui/diff.ts` — otto has the file at `pi-coding-agent/src/modes/interactive/components/diff.ts` (different path!). Otto's `parseDiffLine` at line 9 uses `line.match(/^([+\- ])(\s*\d*)\s(.*)$/)` — the same regex shape (slight char-class formatting diff but functionally identical to upstream's pre-patch `/^([+-\s])(\s*\d*)\s(.*)$/`). Patch rewrites it as manual character checks to avoid catastrophic-backtracking surface area on adversarial diff input. Applies after the path adjustment to otto's diff.ts.

4. `pi-coding-agent/src/core/agent-session.ts` — otto has `parseSkillBlock` at line 103 with the exact upstream pre-patch regex. The patch rewrites it as anchor-search-based string indexing to eliminate the regex-backtracking hazard. Applies cleanly.

5. `pi-coding-agent/src/core/sdk.ts` (otto's path) — upstream patches `gsd-agent-core/src/sdk.ts` `getAttributionHeaders`. Otto's `sdk.ts` lives in pi-coding-agent and has its own attribution-header surface (we saw `openrouter.ai` `.includes(...)` at line 406). The same hostname-spoofing pattern applies. You should mirror the `hasHostname` helper into otto's `sdk.ts` and replace `.includes("opencode.ai")`, `.includes("openrouter.ai")`, `.includes("api.cloudflare.com")`, `.includes("gateway.ai.cloudflare.com")` with strict hostname checks.

6. `pi-agent-core/src/harness/env/nodejs.ts` and `pi-agent-core/src/harness/skills.ts` — otto's `packages/pi-agent-core/src/` does NOT have a `harness/` subtree (it has only `agent-loop`, `agent`, `proxy`, `token-audit`, `types` files). These polyglot-regex-DoS fixes (`/\/+$/.replace`, `/^\/+/.replace`) have no target. SKIP.

# 91a9ca3 — fix(pi-ai): derive Mistral stream message type from request shape

## Target file(s)
- packages/pi-ai/src/providers/mistral.ts

## Divergence
Otto currently imports `ChatCompletionStreamRequestMessages` (plural — matching Mistral SDK v1) at the top of `mistral.ts`, while upstream's pre-commit base was on `ChatCompletionStreamRequestMessage` (singular — SDK v2). Both forms are version-dependent, which is exactly the brittleness upstream's fix removes. The fix replaces the type import with a derived alias `type MistralStreamChatMessage = NonNullable<ChatCompletionStreamRequest["messages"]>[number]`, which is version-agnostic. Otto's call sites at lines 458, 459, and 514 use `ChatCompletionStreamRequestMessages[]` and `ChatCompletionStreamRequestMessages`, and need to be renamed to `MistralStreamChatMessage`.

   ```ts
   type MistralStreamChatMessage = NonNullable<ChatCompletionStreamRequest["messages"]>[number];
   ```

5. Run `pnpm --filter pi-ai exec tsc --noEmit` to verify.

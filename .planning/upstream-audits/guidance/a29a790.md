# a29a790 — fix(coding-agent): drain follow-ups queued during agent_end

## Target file(s)
packages/pi-coding-agent/src/core/agent-session.ts (heavily diverged)
packages/pi-coding-agent/src/core/compaction-orchestrator.ts (otto-cli-only extraction)
(new tests/examples) test/git-merge-and-resolve-extension.test.ts, test/suite/agent-session-queue.test.ts, examples/extensions/git-merge-and-resolve.ts

## Divergence
Major architectural divergence. Upstream uses a synchronous loop `while (await this._handlePostAgentRun()) { await this.agent.continue(); }`, and the fix makes `_handlePostAgentRun` return `this.agent.hasQueuedMessages()` after compaction so the loop drains agent_end-queued follow-ups. otto-cli has NO `_handlePostAgentRun` and no such while-loop — agent_end is event-driven (`_emitExtensionEvent` / `_processAgentEvent` around lines 526-573) and compaction was extracted into `compaction-orchestrator.ts`, which schedules continuations via `_scheduleAutoCompactionFollowup` (setTimeout `agent.continue()`).

The same bug exists in otto-cli: `compactionOrchestrator.checkCompaction(msg)` returns early (e.g. compaction disabled, or below threshold) WITHOUT calling `_scheduleAutoCompactionFollowup`, so messages queued by agent_end extension handlers are NOT drained on the no-compaction path.

## Concrete edits
1. In agent-session.ts, after `await this._compactionOrchestrator.checkCompaction(msg);` (the agent_end branch, ~line 573), add a drain: if no retry/compaction/session-switch occurred and `this.agent.hasQueuedMessages()` is true, schedule `setTimeout(() => this.agent.continue().catch(() => {}), 0)`. Mirror the guard ordering already used in that block (sessionSwitchPending, retry, image-overflow all `return` before reaching this point).
2. Alternatively, have `_scheduleAutoCompactionFollowup` (or a new sibling) be invoked on the non-compaction return paths of `checkCompaction`. Pick whichever keeps the single-drain invariant (avoid double-continue when compaction already scheduled one).
3. Port the queue integration test (agent-session-queue) and optionally the git-merge-and-resolve example.

## Verdict
manual-port — real bug present in otto-cli, but the upstream one-line fix has no 1:1 location due to the event-driven + orchestrator refactor. Port the intent (drain queued follow-ups on the agent_end no-compaction path) and guard against double continuation.

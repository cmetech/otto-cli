# b0e79d9 — fix(installer): prevent handoff timeout and spinner corruption

## Target file(s)
- none (otto-cli has no `scripts/install/` directory)

## Divergence
The commit touches `scripts/install/handoff.js` and `scripts/install/npm-global.js`, plus a new test `src/tests/installer-handoff.test.ts`. Otto-cli has no `scripts/install/` directory and no equivalent custom npm installer flow — otto's `scripts/` contains build/release helpers (`bump-version.mjs`, `validate-pack.js`, `dev-cli.js`, etc.) but no `handoff.js` or `npm-global.js`. Otto installs as a normal npm package without the bespoke post-install spinner/handoff harness that upstream owns. The bug — `spawnSync timeout: 600_000` corrupting the spinner and the old `execFileAsync`-based npm install path lacking output limiting — has no surface in otto's distribution.

1. (none)

## Verdict
Do-not-port. The defect lives in upstream's custom post-install handoff script, which otto-cli does not ship. Caveat: if otto ever introduces a similar interactive post-install handoff, revisit this commit for the `appendLimited` + `spawn`-with-explicit-stdio + timeout-without-blocking-spinner pattern — it is a clean reference design.

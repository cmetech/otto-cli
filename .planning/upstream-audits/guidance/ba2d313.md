# ba2d313 — fix(ai): handle OpenCode Kimi reasoning params

## Target file(s)
upstream: `packages/ai/scripts/generate-models.ts`, `packages/ai/src/models.generated.ts`
otto-cli candidates: `packages/pi-ai/scripts/generate-models.ts`, `packages/pi-ai/src/models/generated/*` (per-provider, not a single models.generated.ts)

## Divergence
Two structural mismatches make this unportable as-is:
1. otto-cli has no `models.generated.ts`. Generated model data lives in per-provider files under `packages/pi-ai/src/models/generated/` (e.g. `opencode-go.ts`), written by `generate-models.ts`. These are build OUTPUT and should not be hand-edited; they regenerate from models.dev + the generator's compat logic.
2. The upstream fix sets `thinkingFormat: "deepseek"`. otto-cli's `OpenAICompletionsCompat.thinkingFormat` type is `"openai" | "zai" | "qwen"` only (packages/pi-ai/src/types.ts:327) — there is NO `deepseek` or `string-thinking` format. otto-cli's `generate-models.ts` also lacks the upstream `applyThinkingLevelMetadata`/kimi-k2.6/string-thinking machinery entirely. The whole reasoning-format branch this commit edits does not exist here.

## Concrete edits
None applicable. Porting would require first importing the entire upstream `deepseek`/`string-thinking` thinkingFormat infrastructure (a much larger, separate effort), which is out of scope for this single fix.

## Verdict
do-not-port — depends on a `thinkingFormat: "deepseek"` mechanism absent from otto-cli's type system and generator; generated model files are regenerated output, not hand-edited. Also overlaps heavily with 4faac05 (same subsystem).

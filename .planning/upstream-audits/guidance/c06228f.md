## Verdict
Manual-port. The underlying defect — npm/npx global installs with `--ignore-scripts` leaving certain hoisted deps as empty placeholders, breaking bundled extensions that require them — applies to otto too: nothing in otto's distribution guarantees `minimatch`, `picomatch`, or `proper-lockfile` survive a `--ignore-scripts` global install. The fix in upstream is two-layered: add to bundleDeps and add aliases in loader. Otto's analog requires both. Caveat: do not blindly add `@modelcontextprotocol/sdk` to bundleDependencies — otto's loader handles MCP via direct static imports instead, which is a different (and arguably cleaner) solution; check whether the static-import path already survives `--ignore-scripts` before duplicating.

# c2f6d35 — fix(bug-1): discardMilestone skips DB cleanup when milestone dir is missing

## Target file(s)

- /Users/coreyellis/code/github.com/cmetech/otto_app/otto-cli/src/resources/extensions/workflow/tests/park-milestone.test.ts

## Divergence
Otto has both files at the matching paths. Upstream's `discardMilestone(basePath, milestoneId)` had an early `if (!mDir || !existsSync(mDir)) return false;` that bailed out before any DB cleanup ran — so if the filesystem milestone was already gone but the DB rows survived, the function lied (returned false, left the DB row stranded). The fix replaces the early return with a "have either filesystem or DB" check: `const hasMilestoneDir = !!mDir && existsSync(mDir); const hasDbMilestone = isDbAvailable() && getMilestone(milestoneId) !== null;` then `if (!hasMilestoneDir && !hasDbMilestone) return false;`. The `rmSync(mDir, ...)` call is guarded `if (hasMilestoneDir && mDir)`. The test addition exercises the "DB rows exist, directory already deleted" path. Otto's `milestone-actions.ts` has the same `discardMilestone` signature using `resolveMilestonePath` and `rmSync` on the resolved directory (verified by symbol presence in otto's workflow tree).

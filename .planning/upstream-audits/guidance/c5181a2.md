# c5181a2 — fix(tui): detect Apple Terminal Shift+Enter

## Target file(s)
packages/pi-tui/src/terminal.ts; new packages/pi-tui/src/native-modifiers.ts; new packages/pi-tui/native/darwin/ (C source + prebuilt .node); test packages/pi-tui/src/__tests__/terminal.test.ts. Upstream build-script changes (scripts/build-binaries.sh) have NO otto-cli equivalent.

## Divergence
Heavily diverged / partly infeasible. otto-cli's `packages/pi-tui/src/terminal.ts` has NO Apple-Terminal handling — its input dispatch is a plain `this.inputHandler(sequence)` (line 164), with no `isAppleTerminalSession`/`normalizeAppleTerminalInput`. There is no `native-modifiers.ts` and no `native/darwin/` tree. otto-cli also has a different native-bindings story (clipboard etc. live under `@otto/native`), and the upstream `scripts/build-binaries.sh` does not exist in otto-cli (release/build scripts were restructured). A cherry-pick will not apply.

## Concrete edits
If the feature is wanted:
- Add the pure-TS helpers to terminal.ts: `isAppleTerminalSession()` (darwin + `TERM_PROGRAM === "Apple_Terminal"`) and `normalizeAppleTerminalInput(data, isAppleTerminal, isShiftPressed)` returning the CSI-u `\x1b[13;2u` for `\r` when shift is pressed.
- In the input dispatch (line ~164), wrap `sequence` through `normalizeAppleTerminalInput`, gated on `isAppleTerminalSession()` and a native shift-modifier probe.
- Decide how to provide the macOS shift probe: either port `native-modifiers.ts` + the `darwin-modifiers.c` napi module and wire prebuilds through otto-cli's native packaging (likely `@otto/native`, NOT `packages/pi-tui/native/...`), or stub the probe to always-false (which degrades to upstream-before-this-fix behavior).
- The pure-TS `normalizeAppleTerminalInput` tests are portable to `packages/pi-tui/src/__tests__/terminal.test.ts`; the `PI_TUI_DISABLE_NATIVE_MODIFIERS` test was removed by 30b3ab3 so omit it.
- Do NOT port the `scripts/build-binaries.sh` hunk; replicate the prebuild-copy step in otto-cli's own build tooling if the native module is adopted.

## Verdict
manual-port — TS normalization helpers are portable, but the native darwin modifier module + binary packaging must be re-homed into otto-cli's native/build system (or stubbed). Verify whether the feature is desired before investing in the C/napi side.

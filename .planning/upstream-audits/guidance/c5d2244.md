# c5d2244 — fix: write context reports under project root

- /Users/coreyellis/code/github.com/cmetech/otto_app/otto-cli/src/resources/extensions/workflow/commands-context.ts

## Divergence
Otto has the parallel commands surface under `workflow/commands/...` and `workflow/commands-context.ts`. The bug: `handleContext(args, ctx)` calls `writeContextChartHtml(process.cwd(), report)` — but in some launch modes `process.cwd()` is not the project root (the user could have launched otto from a subdirectory or via a wrapper), so context reports get written into the wrong location. The fix threads `basePath` through as an optional parameter (defaulting to `process.cwd()` for backwards compatibility) and the core command-handler in `commands/handlers/core.ts` now passes `projectRoot()` when invoking `handleContext` so the explicit root wins over the process cwd. The new test exercises a fixture where the "intended project" directory and `process.cwd()` differ.

## Concrete edits
1. In `workflow/commands-context.ts`:
   - Change the signature `export async function handleContext(args: string, ctx: ExtensionCommandContext): Promise<void>` to `export async function handleContext(args: string, ctx: ExtensionCommandContext, basePath = process.cwd()): Promise<void>`.
   - Inside, replace the `writeContextChartHtml(process.cwd(), report)` call with `writeContextChartHtml(basePath, report)`. If there are other `process.cwd()` references in the function body that should follow the explicit root (the upstream diff likely has just the one), check and update.
2. In `workflow/commands/handlers/core.ts`:
   - Locate the `trimmed === "context" || trimmed.startsWith("context ")` branch that does `await handleContext(...)`. Add a third argument `projectRoot()` so the explicit base path wins. Confirm `projectRoot` is already imported in `core.ts`; if not, add the import.
3. In `workflow/tests/commands-context.test.ts`:
   - Port the `handleContext writes open reports under the command project root` test verbatim. Adjust any fixture paths to use otto's `.otto/` instead of `.gsd/` if the test reads/writes preferences files. The xdg-open binary fixture and `chmodSync`/`mkdtempSync` scaffolding is platform-neutral and transplants cleanly.
4. Run `workflow/tests/commands-context.test.ts`.

Manual-port. The fix is a small signature widening and one call-site change in the command dispatcher. Caveat: if otto's `commands-context.ts` has multiple `process.cwd()` references inside `handleContext` that should now use `basePath`, port all of them — the upstream diff only shows the `--open` branch but the logical intent is "use basePath everywhere the project root is wanted".

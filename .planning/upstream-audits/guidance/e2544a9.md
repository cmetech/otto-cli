## Divergence
Two independent hardening changes, both apply cleanly to otto:

1. `context7/index.ts`: add a `getFirstTextContent(result)` helper (15 lines) that extracts a text block from `result.content[]` defensively, and use it as a fallback in two error-render branches:
   - `Error: ${d?.error ?? getFirstTextContent(result) ?? "unknown"}` instead of `Error: ${d?.error ?? "unknown"}`
   - applied at both the resolve-library tool error path and the docs-fetch tool error path.
   Otto's `src/resources/extensions/context7/index.ts` exists and is structurally close enough to gsd-pi's that this patch should apply with at most trivial offset adjustments. The fix is purely additive (helper + two error-message fallbacks) and risk-free.

2. `search-the-web/native-search.ts`: add a `looksLikeAnthropicModelName(modelName)` helper that matches `claude-*` or `anthropic/claude-*` (case-insensitive after trim). Then in `registerNativeSearchHooks`'s before-request hook, use `payloadModelName` derived from `payload.model` to (a) when `modelSelectFired`, gate `isAnthropic` on `isAnthropicProvider && payloadLooksAnthropic !== false` (so a stale model_select flag plus a concrete non-Claude payload doesn't leak the Anthropic-only `web_search_20250305` tool into an OpenAI Responses request) and (b) when neither `eventModel` nor `modelSelectFired`, use the new helper rather than a raw `claude-` prefix string check. Otto's `src/resources/extensions/search-the-web/native-search.ts` exists; verify the function shape matches before applying.

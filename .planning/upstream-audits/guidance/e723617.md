# e723617 — fix: await extension sendMessage turns

## Target file(s)
- packages/pi-coding-agent/src/core/extensions/types.ts
- packages/pi-coding-agent/src/core/extensions/loader.ts

## Divergence
Upstream `extension-upstream-types.ts` has been renamed/inlined in otto as `packages/pi-coding-agent/src/core/extensions/types.ts`. The `ExtensionAPI.sendMessage` signature lives at otto types.ts:1476 and the `ExtensionActions.sendMessage` signature lives at otto types.ts:1764 — both return `void`. Otto's loader has `sendMessage` at loader.ts:530 implemented as `sendMessage(message, options): void { runtime.sendMessage(message, options); }` (no return). The upstream patch also edits `packages/gsd-agent-core/src/session/agent-session-extensions.ts` (no otto equivalent — skip) and `src/resources/extensions/gsd/bootstrap/register-hooks.ts` (gsd extension only — skip). The new test `packages/pi-coding-agent/test/suite/agent-session-prompt.test.ts` likely depends on gsd test scaffolding; assess on its own merits but the core API-shape change is independently valid.

## Concrete edits
1. In `packages/pi-coding-agent/src/core/extensions/types.ts`, change the `sendMessage` return type on `ExtensionAPI` (around line 1476–1479) from `): void;` to `): Promise<void>;`.
2. In the same file, change the `sendMessage` return type on `ExtensionActions` (around line 1764–1767) from `) => void;` to `) => Promise<void>;`.
3. In `packages/pi-coding-agent/src/core/extensions/loader.ts` around line 530, change the implementation from `sendMessage(message, options): void { runtime.sendMessage(message, options); }` to `sendMessage(message, options): Promise<void> { return runtime.sendMessage(message, options); }`. Add `runtime.assertActive();` ahead of the return if otto's runtime wrapper currently relies on it (the upstream version adds it; verify against otto's loader.ts:528–532 — if `runtime.assertActive()` is missing, add it for parity with upstream).
4. Audit otto callers of `pi.sendMessage(...)` to confirm none drop the now-Promise return on the floor in a way that would re-introduce the race. The compile will fail at sites that previously typed it as `void` and chained anything.
5. Skip the test file at `packages/pi-coding-agent/test/suite/agent-session-prompt.test.ts` unless it lands in otto's test layout cleanly — if so, port it; if it references `gsd-agent-core` types, drop it.

## Verdict
Real fix: extension hosts could fire-and-forget the sendMessage turn, racing the agent. Making the API return a Promise lets callers await delivery. Low risk in otto since the otto extension surface uses the same types.ts/loader.ts contract. The agent-session-extensions hunk is gsd-only and can be ignored.

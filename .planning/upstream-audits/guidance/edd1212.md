# edd1212 — fix(coding-agent): buffer early input before prompt loop

## Target file(s)
`packages/pi-coding-agent/src/modes/interactive/interactive-mode.ts` and `packages/pi-coding-agent/src/modes/interactive/controllers/input-controller.ts`

## Divergence
Same bug/fix is applicable, but the code is restructured so a clean cherry-pick will NOT apply:
- Upstream adds the queue logic directly inside `setupEditorSubmitHandler` (the `onSubmit` handler) in interactive-mode.ts. In otto-cli, `setupEditorSubmitHandler` (line 2403) just delegates to `setupEditorSubmitHandlerController(this)` in `controllers/input-controller.ts`. The actual `onInputCallback` dispatch lives in input-controller.ts (line 85-87: `if (host.onInputCallback) { host.onInputCallback(text); host.editor.addToHistory?.(text); }`). There is currently no `else` branch to queue input when no callback is installed.
- otto-cli `getUserInput()` (line 2707) has no `pendingUserInputs` drain — it immediately returns a Promise that installs `onInputCallback`.
- `private pendingUserInputs: string[]` field does not exist on the class.

## Concrete edits
1. interactive-mode.ts: add field `private pendingUserInputs: string[] = [];` alongside `private onInputCallback?` (near line 340).
2. interactive-mode.ts `getUserInput()` (line 2707): before returning the Promise, add:
   `const queuedInput = this.pendingUserInputs.shift(); if (queuedInput !== undefined) return queuedInput;`
3. input-controller.ts: in the `onInputCallback` dispatch (around line 85), add an `else` branch that pushes to the queue: `if (host.onInputCallback) { host.onInputCallback(text); } else { host.pendingUserInputs.push(text); } host.editor.addToHistory?.(text);`. Ensure the `InteractiveModeHost`/`host` type exposes `pendingUserInputs: string[]` (add to the host interface).
   IMPORTANT: verify which submit path in input-controller.ts corresponds to a "normal prompt" (it has multiple addToHistory call sites at lines 65/96/111/122/132) — only the normal-prompt path (the one that calls onInputCallback) should queue.

## Verdict
manual-port — logic is sound and applies, but the submit handler moved into input-controller.ts and getUserInput diverged; port by hand across both files and extend the host type. Optionally add the upstream startup-input test adapted to otto-cli's controller layout.
